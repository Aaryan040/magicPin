# Vera Merchant AI Assistant Bot

This repository contains the implementation of the Vera AI Assistant Bot for the magicpin AI Challenge. The bot is implemented in Python using FastAPI, fully satisfying the requirements of the 4-context framework and the multi-turn conversation specifications.

## Key Features

1. **Fully Deterministic Composition**: Message generation uses a rule-based template router mapping all 24 trigger kinds. This guarantees:
   - **Zero Latency (< 1ms)**: No external LLM queries during runtime.
   - **No Hallucinations**: Perfect specificity, strictly using data variables directly from trigger, category, merchant, and customer contexts.
   - **Tone Consistency**: Clinically appropriate peer salutations for dentists, motivational tone for gyms, trustworthy voice for pharmacies, etc.

2. **Advanced Multi-Turn Logic**:
   - **Auto-Reply Loop Control**: Automatically counts and mitigates consecutive merchant auto-replies, backing off for 24h or gracefully ending the thread.
   - **Intent Transitions**: Switches immediately from pitching to confirmation/action mode upon merchant affirmation (`"let's do it"`, `"confirm"`).
   - **Redirection & Graceful Ends**: Handles out-of-scope queries (e.g. GST filing) politely before steering the conversation back to business growth, and gracefully terminates on opt-outs.

3. **Browser Testing & Fallbacks**: Exposes base `/` and `/v1` paths and friendly GET fallbacks for `/v1/context`, `/v1/tick`, and `/v1/reply` to prevent browser `405 Method Not Allowed` errors during manual testing.

## How to Run Locally

### 1. Install Dependencies
Ensure you have `fastapi` and `uvicorn` installed:
```bash
pip install fastapi uvicorn pydantic
```

### 2. Start the Bot Server
Run Uvicorn to host the app on port 8080:
```bash
uvicorn bot:app --host 127.0.0.1 --port 8080
```
- Open `http://127.0.0.1:8080/v1` in your browser to inspect available routes.
- Open `http://127.0.0.1:8080/v1/healthz` to see the health and count of loaded contexts.

### 3. Run the Local Judge Simulator
To run the evaluation harness locally:
```bash
python judge_simulator.py
```

## How to Share / Submit the Project

To submit the project to the magicpin evaluation harness, you need to expose your local server to the public internet.

### Expose Using ngrok
1. Download and install [ngrok](https://ngrok.com/).
2. Run the following command in a separate terminal while your bot server is running:
   ```bash
   ngrok http 8080
   ```
3. Copy the public forwarding HTTPS URL generated by ngrok (e.g., `https://xxxx-xxxx.ngrok-free.app`).
4. Paste this public base URL into the magicpin submission portal. The harness will query your public URL at `https://xxxx-xxxx.ngrok-free.app/v1/*` to test context pushes, ticks, and reply cycles.
